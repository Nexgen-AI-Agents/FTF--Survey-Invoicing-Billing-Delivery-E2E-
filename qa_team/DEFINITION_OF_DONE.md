# Definition of Done — FTF Agentic AI OS

A sprint is **DONE** only when ALL of the following are true.
No exceptions. No partial done.

---

## Development Complete

- [ ] All tasks in the sprint file are checked off
- [ ] All agent `.py` files written and in `code/sprint_NN/agents/`
- [ ] All test files written and in `code/sprint_NN/tests/`
- [ ] All unit tests pass with zero errors
- [ ] No `TODO` or `FIXME` stubs in production code
- [ ] Sprint `README.md` describes what was built, how to run it, how to test it
- [ ] `dev_team/developer_review.md` updated if new learnings exist
- [ ] PR Checklist (`dev_team/PR_CHECKLIST.md`) fully signed off

---

## Code Review Complete

- [ ] Junior Dev self-check passed
- [ ] Senior Dev review passed (logic, standards, edge cases)
- [ ] Dev Manager final approval given

---

## QA Complete

- [ ] Junior QA passed (happy path, functional)
- [ ] Senior QA passed (edge cases, integration, security, regression)
- [ ] Manager QA sign-off given
- [ ] All test cases in `qa_team/test_cases/sprint_NN_test_cases.md` pass
- [ ] No BLOCKER or CRITICAL issues open in `issues/issue.md`
- [ ] `qa_team/QA_learning.md` updated

---

## Documentation & Tracking Complete

- [ ] Sprint file Completion Brief filled in
- [ ] `sprints/index.md` status updated to ✅ Complete
- [ ] `client_progress_tracker.md` updated
- [ ] `issues/issue.md` — all sprint issues resolved or explicitly deferred
- [ ] `CHANGELOG.md` entry added for this sprint
- [ ] `memory.md` Sprint Briefs section updated

---

## Released

- [ ] All files committed and pushed to GitHub
- [ ] GitHub remote confirmed — no pending local-only commits
- [ ] If milestone sprint (10, 11, 12): Prateek sign-off received before prod deploy

---

> If any box is unchecked, the sprint is NOT done. Status stays 🔄 In Progress.
